1. Platform Overview

ncrptx is a digital asset platform built as a static website deployed on GitHub Pages with a custom domain (ncrptx.com). The platform consists of 7 interconnected modules:

  • Landing Page — Corporate homepage with services, team, and contact
  • Wallet — 3-tier KYC onboarding with unique customer IDs
  • Onramp — Fiat-to-crypto buy widget (Onramper-inspired)
  • Trade — Binance-inspired trading UI with live mock data
  • Pay by Crypto — Crypto payment interface
  • Gateway — Merchant crypto payment gateway
  • Partner Projects — Partner tier showcase
All pages share a unified navigation bar injected via assets/nav.js with the ncrptx SVG logo and 5 main feature links.

4. Wallet System & KYC

The wallet onboarding is a 4-step flow that adapts based on the user's country of residence.

4-Step Flow

  1. Country Selection — User selects country → system detects KYC group
  2. Wallet Type — User selects custodial or non-custodial (skipped for Group A)
  3. KYC Form — Form fields shown based on compliance group
  4. Success — Unique Customer ID generated and displayed

5. Country Groups

Country groups are defined in wallet/countries.js and can be edited by the backend team.

GroupKYCWallet OptionsDescription
Group A Full KYC Mandatory Regulated Custodial only US, EU, UK, CH, JP, SG, AU, CA, and other regulated jurisdictions
Group B Optional Custodial or Non-Custodial TR, NG, KE, PH, TH, VN, ID, UA, and other emerging markets
Group C None Non-Custodial or Custodial All remaining countries not in Group A or B
⚠️ Backend note: Edit COUNTRY_GROUPS.A and COUNTRY_GROUPS.B arrays in wallet/countries.js to update compliance requirements per country. Group C is automatic (all non-A/B countries).

6. Customer ID System

Every customer receives a unique ID at account creation. The format is:

NCX-{TIMESTAMP_BASE36}-{RANDOM_6CHARS}
Example: NCX-LKFJ3A-X7QR2M

Design Notes for Backend

  • Customer IDs are globally unique (timestamp + random component)
  • IDs are designed to support P2P transfers between customers
  • IDs support sub-wallet hierarchy: parent customer can create child wallets linked to their ID
  • Sub-wallet format recommendation: NCX-PARENTID-SUB01, NCX-PARENTID-SUB02, etc.
  • Parental control flag: is_supervised: true on sub-wallets

Recommended Database Schema (Backend)

7. Trading Interface

The trading UI (ncrptx.com/trade/) is a full Binance-inspired interface with mock data.

  • Order Book — Live ask/bid simulation with depth bars
  • Price Chart — Chart.js line chart with volume bars, 7 timeframes
  • Order Form — Buy/Sell with Limit, Market, Stop-Limit order types
  • 10 Trading Pairs — BTC, ETH, SOL, BNB, XRP, ADA, DOGE, DOT, LINK, AVAX
  • Live Price Feed — Prices tick every 800ms (simulated)
  • Open Orders Panel — Orders placed appear and can be cancelled
⚠️ All trading data is simulated. Backend integration required for real order execution (WebSocket for live prices, REST API for order placement).

8. Onramp Widget

The onramp page (ncrptx.com/onramp/) features an interactive fiat-to-crypto widget.

  • Supports Buy, Sell, Swap tabs
  • Currency cycling: USD, EUR, GBP, TRY, CHF
  • Crypto cycling: BTC, ETH, SOL, BNB, USDT, USDC
  • Mock exchange rates for UI demonstration
  • 130+ payment methods showcase
💡 To activate real payments: integrate Transak or Onramper API. Both have free sandbox access. Replace mock rates in onramp/main.js → RATES object with live API calls.
